## 1. O que está SAUDÁVEL (não mexer)

Confirmado por leitura do código, não por suposição:

**Infraestrutura técnica**
- `robots.ts` correto: `allow: "/"`, `disallow: "/api/"`, e regra explícita liberando GPTBot, ClaudeBot, PerplexityBot, Google-Extended e CCBot. Sitemap declarado.
- Root `layout.tsx`: `metadataBase`, template de título `%s — ElectioLab`, `lang="pt-BR"`, verificação Google Search Console.
- Grupo `(dashboard)` exporta `robots: { index: false, follow: true }` — `follow: true` deliberado preserva link equity para as páginas públicas. Correto.
- **SSR confirmado**: em `pesquisas-presidenciais-2026/page.tsx` e `candidato/[slug]/page.tsx`, as porcentagens chegam do Supabase **no servidor** e estão no HTML inicial. O Googlebot vê os números sem executar JS. (Só o `DriftChart` é client-side, mas os dados textuais já estão no HTML.)
- ISR bem calibrado: `revalidate=3600` nas páginas de dado; `force-dynamic` onde depende de `searchParams` (candidatos, comparar, mapa); `dynamicParams=false` nas pré-renderizadas (eleicoes/[uf], pesquisas-senador, eleicao-2018/2022).
- Canonical próprio em **64 de 83** `page.tsx`. As rotas dinâmicas (candidato, instituto, eleicoes/[uf]) geram canonical absoluto com o slug/UF correto — nenhuma usa placeholder errado.

**Structured data**
- `WebSite` + `SearchAction` (sitelinks searchbox) e `Organization` com `sameAs` na home.
- `Dataset` (home, presidencial, eleicoes/[uf]) com `license` CC-BY, `distribution` apontando para a API, `variableMeasured`, `alternateName` e `citation` — padrão raro e forte para citação por IA.
- `FAQPage` em ~43 páginas, `Person` nos candidatos, `NewsArticle` nos relatórios, `Product` + `Offer` (BRL) em `/precos` para rich snippet de preço.

**On-page**
- Títulos únicos e keyword-first; as 27 páginas de governador **não** são near-duplicate (variam candidato, placar, instituto, mês: "PR — Sergio Moro Lidera com 51%", "MG — Cleitinho lidera").
- H1 único, textual e indexável em todas as páginas; a home (dashboard-like) tem H1 real + parágrafo `sr-only` reforçando a keyword longa.
- Cluster editorial com profundidade real (~900–1.200 palavras por spoke, exemplos históricos reais) e internal linking via `LeiaTabem`.

**GEO**
- Acesso total a crawlers de IA; `/imprensa` com "como citar"; `llms.txt` de 10KB bem estruturado com instrução de citação consistente em 3 lugares (llms.txt, home, imprensa).

## 2. Achados priorizados (impacto × esforço)

### 🔴 CRÍTICO

#### C1 — Canonical com query string em `/comparar` vaza equity
`src/app/comparar/page.tsx:148-151`. Como é `force-dynamic`, o `generateMetadata` está montando o canonical **com** os parâmetros de busca, criando uma canonical única por combinação de candidatos.

```ts
// ANTES (errado) — cada combinação vira uma canonical distinta
canonical: "https://electiolab.com/comparar" +
  (slugs.length ? `?${slugs.map((s,i) => `${"abc"[i]}=${s}`).join("&")}` : "")

// DEPOIS — canonical limpa, sempre a mesma
alternates: { canonical: "https://electiolab.com/comparar" },
```
**Impacto:** alto · **Esforço:** baixo (1 linha). O `openGraph.url` já está correto.

#### C2 — Canibalização `/eleicoes/{uf}` × `/eleicoes-governador-{uf}-2026`
As duas famílias (27 + 27) miram a mesma intenção "pesquisas governador {UF} 2026". Cada uma tem canonical para si mesma; não há cross-canonical. A `/eleicoes/{uf}` já linka para a estática via `govUrl`, mas isso não resolve a competição.

A página **estática** é a mais rica (FAQ longo, história do estado, análise) e tem a intenção primária de pesquisa. A **dinâmica** é um hub (governador + senador + deputado). Duas saídas:
- **Recomendada (consolidar):** `/eleicoes/{uf}` aponta canonical para a estática.
  ```ts
  // eleicoes/[uf]/page.tsx — generateMetadata()
  alternates: { canonical: `https://electiolab.com/eleicoes-governador-${uf}-2026` },
  ```
- **Alternativa (diferenciar):** manter ambas indexáveis, mas reposicionar `/eleicoes/{uf}` como "hub eleitoral completo do estado" (governador + senador + câmara) e a estática como "corrida de governador", com títulos/H1 distintos.

**Impacto:** alto · **Esforço:** médio.

### 🟠 ALTO

#### A1 — `instituto/[slug]` usa `AggregateRating` indevidamente (risco de penalidade)
`src/app/instituto/[slug]/page.tsx:194-199`. O `reliability_score` do instituto vira `ratingValue` (0–100) e `reviewCount = polls.length`. `AggregateRating` pressupõe **avaliações de usuários**; aqui é um score interno auto-atribuído com "reviews" que são pesquisas, não avaliações. Isso é exatamente o tipo de marcação que o Google classifica como *spammy structured markup* (risco de ação manual) — e star ratings auto-atribuídos costumam ser ignorados ou punidos.

```jsonc
// ANTES — AggregateRating auto-atribuído (risco)
"aggregateRating": {
  "@type": "AggregateRating",
  "ratingValue": reliabilityPct, "bestRating": 100, "worstRating": 0,
  "reviewCount": polls.length
}

// DEPOIS (opção 1) — Review autoral explícito do ElectioLab
"review": {
  "@type": "Review",
  "author": { "@type": "Organization", "name": "ElectioLab" },
  "reviewRating": { "@type": "Rating", "ratingValue": reliabilityPct, "bestRating": 100 },
  "reviewBody": "Score de acurácia histórica vs. resultado TSE, calculado pelo ElectioLab."
}

// DEPOIS (opção 2, mais segura) — PropertyValue, sem semântica de review
"additionalProperty": {
  "@type": "PropertyValue",
  "name": "Score de acurácia ElectioLab",
  "value": reliabilityPct, "maxValue": 100,
  "description": "Erro médio absoluto vs. TSE nas últimas eleições"
}
```
**Impacto:** alto (risco) · **Esforço:** baixo.

#### A2 — `/auth/login` indexável, sem noindex, herdando canonical da home
`src/app/auth/login/page.tsx`. Está fora do grupo `(dashboard)`, então **herda o canonical da home** (`https://electiolab.com`) do root layout e é indexável → o Google pode tratar o login como duplicata parcial da home.
```ts
// adicionar
export const metadata: Metadata = {
  title: "Entrar — ElectioLab",
  robots: { index: false, follow: false },
};
```
**Impacto:** alto · **Esforço:** baixo. (Revisar também `newsletter/confirmar`, `newsletter/cancelado`, `embed/eleicao/[id]` — confirmar noindex onde fizer sentido.)

#### A3 — Canibalização Home × `/pesquisas-presidenciais-2026`
Títulos quase idênticos ("— Média Agregada | ElectioLab") competindo pelo mesmo termo. Hierarquizar a intenção:
- **Home** = termo guarda-chuva: `ElectioLab — Pesquisas Eleitorais 2026 (Presidente, Governador, Senado)`.
- **/pesquisas-presidenciais-2026** = intenção específica: `Pesquisas para Presidente 2026 — Lula × Bolsonaro × Tarcísio, Média ao Vivo`.

A home deve linkar explicitamente para a página presidencial como "ver análise completa". **Impacto:** alto · **Esforço:** médio. (Confirmado que **não existe** rota `/pesquisas-presidenciais` sem sufixo — esse canibal não está presente, ótimo.)

#### A4 — E-E-A-T fraco para YMYL eleitoral (autoria + credenciais)
Conteúdo eleitoral é YMYL ("Your Money or Your Life") — o Google é rígido com quem está por trás. Gaps:
- **Sem byline visível** em artigos do glossário e nos relatórios semanais — o autor existe só no JSON-LD invisível. Adicionar "Por Luiz Lessa · {data}" legível acima/abaixo do H1.
- **Bio do fundador genérica** em `/sobre` ("desenvolvedor independente apaixonado por dados públicos"). "Apaixonado" não é credencial. Trocar por especificidade mensurável: formação, projetos com dados públicos, "sistema que processou N pesquisas de 3 eleições".
- **Sem link de registro TSE** das pesquisas citadas (equivale ao DOI de um estudo). Adicionar ao menos 1 link PesqEle/TSE por página de estado.
- **Sem política de correções**. Adicionar linha "Correções e atualizações" no rodapé / `/metodologia`.

**Impacto:** alto · **Esforço:** médio.

#### A5 — Frescor: dado defasado em estados pequenos + `dateModified` hardcoded
- As 27 páginas de governador têm `datePublished: "2026-04-01"` e `dateModified: "2026-04-23"` **hardcoded** — 27 páginas com a mesma data é sinal técnico de geração em lote e degrada o frescor exatamente no tipo de conteúdo que o Google mais monitora.
- Estados pequenos citam pesquisas de **set/2025 (AC)** e **dez/2025 (RR)** como "mais recente" — 6–9 meses de defasagem, sem aviso.

```ts
// ANTES
dateModified: "2026-04-23"
// DEPOIS — derivar do snapshot ao vivo já carregado
dateModified: govSnapshot?.publication_date ?? "2026-04-23"
```
+ badge condicional quando a pesquisa mais recente tiver >90 dias: *"Dados com mais de 90 dias — poucos institutos cobrem este estado neste período."*
**Impacto:** alto (frescor + confiança YMYL) · **Esforço:** baixo-médio.

#### M4 — Respostas citáveis (answer-first) enterradas — GEO
As páginas-resposta (`pesquisas-presidenciais-2026`, `quem-vence-no-segundo-turno`, governadores) começam com hero de marketing; o primeiro número citável está dentro de um componente de barras (`<div>`), não numa frase auto-contida. IAs extraem os primeiros 40–60 caracteres de cada bloco.
```
// Inserir logo abaixo do H1, com dados que JÁ estão no servidor (averages1t):
"Pela média ponderada do ElectioLab em {data} ({n} pesquisas, {totalN} entrevistados):
Lula 36,8% (±2,1), Bolsonaro 30,4% (±2,3), Tarcísio 18,2% (±1,8). No 2º turno, {cenário}

```
A página `/instituto-mais-acurado-eleicoes-brasil` ("A resposta curta: Datafolha tem o melhor histórico…") já é o **modelo certo** — replicar o padrão. **Impacto:** médio-alto (GEO) · **Esforço:** baixo-médio.

#### M5 — Diferencial subutilizado nas páginas de maior tráfego
Metodologia aberta + ranking de acurácia por instituto é o ângulo mais defensável (nem Poder360 nem Quaest têm) — mas nas 27 páginas de governador e na presidencial aparece só em letra miúda no rodapé. Adicionar um bloco curto **"Por que confiar nesta média"** (link para `/metodologia` + ranking de acurácia) em cada página de pesquisa: vira sinal de E-E-A-T repetido nas URLs de maior tráfego e combustível de citação para IA. **Impacto:** médio-alto · **Esforço:** baixo-médio.

#### M6 — Higiene de sitemap e cobertura senatorial

- `/api` está no sitemap; confirmar via `curl -I https://electiolab.com/api` que o Vercel não o redireciona para `/api/` (bloqueado pelo `disallow: "/api/"`).
- `pesquisas-senador/[uf]` é `dynamicParams=false` com só 8 UFs no `generateStaticParams` e no sitemap → as outras 19 UFs dão **404**. Auditar links internos para garantir que nenhum aponta para UF não coberta; decidir se amplia o array.
- Quando sair a semana 23, mudar `changeFrequency` da semana-22 de `"weekly"` para `"never"`.

**Impacto:** médio · **Esforço:** baixo.

## 3. Gap competitivo — a oportunidade estrutural

Pesquisa sobre Poder360 e Quaest (fontes ao final). O ElectioLab **domina** a cauda informacional (margem de erro, empate técnico, "por que institutos divergem") que nenhum concorrente cobre bem, e é o **único** com metodologia aberta + ranking de acurácia + API pública. Mas há um buraco de cabeça de cauda:

| Tipo de conteúdo | Poder360 | Quaest | ElectioLab |
|---|---|---|---|
| Agregador de intenção de voto | ✅ | parcial | ✅ (forte) |
| **Aprovação/desaprovação de governo (agregada)** | ✅ série histórica | ✅ | ❌ **GAP** |
| **Rejeição de candidatos (agregada)** | ✅ | ✅ | ❌ **GAP** |

| Metodologia aberta + ranking de acurácia | ❌ | ❌ | ✅ **só ElectioLab** |

**Queries de cabeça sem cobertura:** "aprovação do governo Lula / aprovação Lula hoje", "desaprovação Lula", "rejeição Bolsonaro 2026", "aprovação governador {UF}". São recorrentes (mensais) e o ElectioLab já tem o motor de média ponderada para agregá-las — seria o mesmo motor sobre um novo tipo de dado, e o lugar onde a metodologia aberta vira vantagem real. **Este é o maior retorno estrutural do roadmap.**
